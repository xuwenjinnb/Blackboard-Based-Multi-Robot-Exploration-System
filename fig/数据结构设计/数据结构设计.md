# 多小车协作巡检仿真系统数据结构设计

本文档按照组件职责重新整理系统数据结构。系统采用 Redis 黑板保存运行状态，Controller Worker、Navigator Worker、Robot Fleet Worker 不直接调用彼此内部方法，而是通过 Redis 中的车辆、前沿点、任务、导航请求、路径规划、地图和心跳等结构交换数据。表格中的“表”表示逻辑数据表或逻辑数据对象。

## 1）组件使用的数据结构

### 1.1 导航器 Navigator Worker

导航器负责路径规划。它主要读取 `PENDING` 状态的导航请求，读取地图、车辆和任务状态来规划路径，然后把规划结果写入路径规划表。成功规划后，黑板会同步把路径写入任务的 `pathQueue`，供小车执行。

#### 1.1.1 导航器主要读写结构

| 数据结构 | Redis Key | 读/写 | 导航器中的用途 |
| --- | --- | --- | --- |
| RuntimeConfigInfo | `inspection:runtime` | 读 | 判断导航算法，如 `baseline` 或 `cbs`，以及导航器数量 |
| NavigationRequestInfo | `inspection:navigation_requests` | 读/写 | 领取 `PENDING` 请求，标记为 `PLANNING`，规划完成后更新为 `SUCCESS` 或 `FAILED` |
| NavigationPlanInfo | `inspection:navigation_plans` | 写 | 保存 A*、时空 A* 或 CBS 输出的路径规划结果 |
| TaskInfo | `inspection:tasks` | 读/写 | 读取已有任务路径构造预约表；成功规划后写入 `pathQueue` |
| VehicleInfo | `inspection:vehicles` | 读 | 获取车辆当前位置，构造避让预约或 CBS 固定约束 |
| MapInfo / MapChunkInfo / MapCellInfo | `inspection:map:meta`、`inspection:map:chunks` | 读 | 读取地图尺寸、版本和格子状态，用于路径规划 |
| ComponentHeartbeatInfo | `inspection:heartbeats` | 写 | 上报导航器 `READY` 或 `BUSY` 状态 |

#### 1.1.2 导航请求表 NavigationRequestInfo

存储位置：`inspection:navigation_requests`，Redis Hash，field 为 `requestId`，value 为导航请求 JSON。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| requestId | varchar | 32 | 主键，不能为空 | 导航请求编号 |
| vehicleId | varchar | 32 | 外键，不能为空 | 请求规划的小车编号 |
| taskId | varchar | 32 | 外键，不能为空 | 所属任务编号 |
| start | json | Max | 不能为空 | 路径起点坐标，如 `{"x": 2, "y": 2}` |
| goal | json | Max | 不能为空 | 路径终点坐标，即任务目标点 |
| mapVersion | int | 10 | 不能为空 | 创建请求时的地图版本 |
| priority | int | 3 | 不能为空 | 请求优先级，普通任务默认 5，重规划任务为 8 |
| avoidVehicles | boolean | 1 | 不能为空 | 是否避让其他小车，普通请求默认为 true |
| status | varchar | 16 | 不能为空 | 请求状态，包含 PENDING、PLANNING、SUCCESS、FAILED |
| assignedNavigatorId | varchar | 32 | 可为空 | 领取请求的导航器编号 |
| createdAt | bigint | 13 | 不能为空 | 请求创建时间戳 |
| updatedAt | bigint | 13 | 不能为空 | 请求更新时间戳 |

#### 1.1.3 路径规划表 NavigationPlanInfo

存储位置：`inspection:navigation_plans`，Redis Hash，field 为 `planId`，value 为路径规划 JSON。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| planId | varchar | 32 | 主键，不能为空 | 路径规划编号，由黑板写入时生成 |
| requestId | varchar | 32 | 外键，不能为空 | 对应导航请求编号 |
| taskId | varchar | 32 | 外键，不能为空 | 对应任务编号 |
| vehicleId | varchar | 32 | 外键，不能为空 | 对应小车编号 |
| start | json | Max | 不能为空 | 规划起点 |
| goal | json | Max | 不能为空 | 规划终点 |
| path | json | Max | 可为空 | 路径点列表；失败时为空列表 |
| cost | int | 8 | 不能为空 | 路径列表长度，代码中取 `len(path)` |
| distance | int | 8 | 不能为空 | 移动距离，代码中取 `len(path)-1` |
| estimatedTime | int | 8 | 不能为空 | 预计执行时间片数，当前与 `distance` 一致 |
| mapVersion | int | 10 | 不能为空 | 规划使用的地图版本 |
| status | varchar | 16 | 不能为空 | 规划状态，包含 SUCCESS、NO_PATH |
| failReason | varchar | 200 | 可为空 | 规划失败原因，仅失败时写入 |
| planner | varchar | 20 | 不能为空 | 使用的规划器，如 astar、st_astar、cbs |
| createdBy | varchar | 32 | 不能为空 | 生成路径的导航器编号 |
| createdAt | bigint | 13 | 不能为空 | 规划生成时间戳 |

#### 1.1.4 路径点结构 PathStepInfo

逻辑结构：嵌套在 `NavigationPlanInfo.path` 和 `TaskInfo.pathQueue` 中。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| stepIndex | int | 8 | 主键字段，不能为空 | 路径步序号 |
| position | json | Max | 不能为空 | 当前路径点坐标，包含 `x`、`y` |
| heading | int | 3 | 不能为空 | 到达该路径点时的小车朝向 |
| expectedTimeSlot | int | 8 | 可为空 | 预计时间片；时空 A* 和 CBS 使用 |
| action | varchar | 8 | 不能为空 | 动作类型，包含 MOVE、WAIT |

#### 1.1.5 时空预约结构 ReservationInfo

逻辑结构：运行时临时结构，不直接持久化到 Redis。时空 A* 和 CBS 会从 `tasks.pathQueue`、`tasks.currentStepIndex`、`vehicles.pose` 中构造预约表。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| timeSlot | int | 8 | 主键字段，不能为空 | 时间片编号 |
| reservedCells | set/json | Max | 可为空 | 该时间片已被占用的格子集合 |
| edgeReservations | set/json | Max | 可为空 | 已被占用的时空边集合，元素形如 `(from, to, timeSlot)` |
| sourceTaskId | varchar | 32 | 可为空 | 预约来源任务编号 |
| sourceVehicleId | varchar | 32 | 可为空 | 预约来源车辆编号 |

### 1.2 控制器 Controller Worker

控制器负责任务分配。它读取空闲车辆、OPEN 前沿点、地图和已有任务状态，调用 nearest 或 greedy 策略生成分配决策，然后创建任务和导航请求。

#### 1.2.1 控制器主要读写结构

| 数据结构 | Redis Key | 读/写 | 控制器中的用途 |
| --- | --- | --- | --- |
| RuntimeConfigInfo | `inspection:runtime` | 读 | 判断任务分配策略，如 nearest、greedy、low_mdp |
| VehicleInfo | `inspection:vehicles` | 读/写 | 读取空闲车辆，分配任务后写入 `currentTaskId` |
| FrontierInfo | `inspection:frontiers` | 读/写 | 读取 `OPEN` 前沿点，分配后更新为 `ASSIGNED` |
| TaskInfo | `inspection:tasks` | 读/写 | 创建任务；对需要重规划的 PENDING 任务补发导航请求 |
| NavigationRequestInfo | `inspection:navigation_requests` | 写 | 为任务创建待规划导航请求 |
| MapInfo / MapChunkInfo / MapCellInfo | `inspection:map:meta`、`inspection:map:chunks` | 读 | nearest 策略计算可达性，greedy 策略计算未知收益 |
| ComponentHeartbeatInfo | `inspection:heartbeats` | 写 | 上报控制器 `BUSY` 或 `READY` 状态 |

#### 1.2.2 前沿点信息表 FrontierInfo

存储位置：`inspection:frontiers`，Redis Hash，field 为 `frontierId`，value 为前沿点 JSON。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| frontierId | varchar | 32 | 主键，不能为空 | 前沿点编号 |
| position | json | Max | 不能为空 | 前沿点坐标，包含 `x`、`y` |
| unknownGain | int | 6 | 不能为空 | 该前沿点扫描半径内的未知格子数量 |
| discoveredBy | varchar | 32 | 不能为空 | 发现该前沿点的小车编号 |
| status | varchar | 16 | 不能为空 | 前沿点状态，包含 OPEN、ASSIGNED、VISITED、CLOSED |
| timestamp | bigint | 13 | 不能为空 | 发现时间戳 |
| updatedAt | bigint | 13 | 可为空 | 状态刷新时间戳 |

#### 1.2.3 任务信息表 TaskInfo

存储位置：`inspection:tasks`，Redis Hash，field 为 `taskId`，value 为任务 JSON。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| taskId | varchar | 32 | 主键，不能为空 | 任务编号 |
| vehicleId | varchar | 32 | 外键，不能为空 | 执行任务的小车编号 |
| target | json | Max | 不能为空 | 任务目标点坐标 |
| frontierId | varchar | 32 | 外键，不能为空 | 任务对应前沿点编号 |
| status | varchar | 16 | 不能为空 | 任务状态，包含 PENDING、PLANNED、RUNNING、DONE、FAILED、CANCELLED |
| planId | varchar | 32 | 可为空 | 当前任务对应的路径规划编号 |
| pathQueue | json | Max | 可为空 | 小车待执行路径队列 |
| currentStepIndex | int | 8 | 不能为空 | 当前执行到的路径步数 |
| replanCount | int | 4 | 不能为空 | 重新规划次数 |
| createdAt | bigint | 13 | 不能为空 | 任务创建时间戳 |
| updatedAt | bigint | 13 | 不能为空 | 任务更新时间戳 |

#### 1.2.4 分配决策结构 AssignmentDecision

逻辑结构：Controller 策略运行时输出，不直接存入 Redis。控制器根据该结构创建 `TaskInfo` 和 `NavigationRequestInfo`。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| vehicle_id | varchar | 32 | 不能为空 | 被分配的小车编号 |
| frontier_id | varchar | 32 | 不能为空 | 被分配的前沿点编号 |
| priority | int | 3 | 不能为空 | 生成导航请求时使用的优先级，默认 5 |
| score | float | 8 | 不能为空 | 策略内部评分，nearest 中为负路径距离，greedy 中为收益评分 |

### 1.3 小车 Robot Fleet Worker

小车负责执行路径、扫描地图、上传地图 patch、发现 frontier，并更新自身状态。普通模式下小车读取任务路径执行；`low_mdp` 模式下小车直接根据局部地图自主选择下一步。

#### 1.3.1 小车主要读写结构

| 数据结构 | Redis Key | 读/写 | 小车中的用途 |
| --- | --- | --- | --- |
| VehicleInfo | `inspection:vehicles` | 读/写 | 读取和更新小车位置、朝向、电量、状态、当前任务 |
| TaskInfo | `inspection:tasks` | 读/写 | 获取当前任务，推进 `currentStepIndex`，完成或阻塞时更新任务状态 |
| NavigationPlanInfo / PathStepInfo | `inspection:navigation_plans`、`tasks.pathQueue` | 读 | 按规划路径执行下一步 |
| MapInfo / MapChunkInfo / MapCellInfo | `inspection:map:meta`、`inspection:map:chunks` | 读/写 | 上传扫描 patch，更新格子状态和地图版本 |
| FrontierInfo | `inspection:frontiers` | 写 | 扫描后保存新发现的前沿点 |
| ComponentHeartbeatInfo | `inspection:heartbeats` | 写 | 上报小车 `READY` 或 `BUSY` 状态 |
| EventLogInfo | `inspection:events` | 写 | 记录地图更新、阻塞、任务完成等事件 |

#### 1.3.2 小车信息表 VehicleInfo

存储位置：`inspection:vehicles`，Redis Hash，field 为 `vehicleId`，value 为小车 JSON。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| vehicleId | varchar | 32 | 主键，不能为空 | 小车编号 |
| pose | json | Max | 不能为空 | 小车位姿，包含 `position` 和 `heading` |
| speed | float | 8 | 不能为空 | 小车速度，仿真中默认 1.0 |
| battery | int | 3 | 不能为空 | 小车电量，移动时递减 |
| status | varchar | 16 | 不能为空 | 小车状态，包含 IDLE、MOVING、SCANNING、BLOCKED |
| currentTaskId | varchar | 32 | 可为空 | 当前执行任务编号 |
| currentPlanId | varchar | 32 | 可为空 | 当前执行路径规划编号 |
| currentStepIndex | int | 8 | 不能为空 | 当前执行到的路径步数 |
| updatedAt | bigint | 13 | 不能为空 | 小车状态更新时间戳 |

#### 1.3.3 地图 Patch 结构 MapPatchInfo

逻辑结构：小车扫描后上传的临时数据对象，黑板合并后更新 `MapInfo`、`MapChunkInfo` 和 `MapCellInfo`。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| patchId | varchar | 32 | 主键，不能为空 | 地图 patch 编号 |
| vehicleId | varchar | 32 | 外键，不能为空 | 上传 patch 的小车编号 |
| baseMapVersion | int | 10 | 不能为空 | patch 基于的地图版本 |
| cells | json | Max | 不能为空 | 本次扫描得到的格子列表 |
| timestamp | bigint | 13 | 不能为空 | patch 上传时间戳 |

#### 1.3.4 地图格子表 MapCellInfo

逻辑结构：嵌套在地图分块或地图快照中。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| x | int | 4 | 主键字段，不能为空 | 格子 X 坐标 |
| y | int | 4 | 主键字段，不能为空 | 格子 Y 坐标 |
| state | varchar | 16 | 不能为空 | 格子状态，包含 UNKNOWN、FREE、VISITED、OBSTACLE、RESERVED |
| confidence | float | 8 | 不能为空 | 感知置信度 |
| updatedBy | varchar | 32 | 可为空 | 最后更新该格子的小车编号或组件编号 |
| updatedAt | bigint | 13 | 不能为空 | 格子更新时间戳 |

| navigationPlans | json | Max | 可为空 | 当前帧路径规划结果 |
| events | json | Max | 可为空 | 当前帧系统事件 |

## 4）数据结构关系说明

| 关系 | 说明 |
| --- | --- |
| VehicleInfo 1:N TaskInfo | 一辆小车可以在不同时间执行多个任务 |
| FrontierInfo 1:1 TaskInfo | 一个前沿点被分配后生成一个巡检任务 |
| TaskInfo 1:N NavigationRequestInfo | 一个任务可能因为失败或阻塞产生多次导航请求 |
| NavigationRequestInfo 1:1 NavigationPlanInfo | 一个导航请求成功或失败后生成一个规划结果 |
| NavigationPlanInfo 1:N PathStepInfo | 一条路径规划包含多个路径点 |
| VehicleInfo 1:1 ComponentHeartbeatInfo | 每辆小车通过心跳上报当前运行状态 |
| UserInfo 1:N SessionInfo | 一个用户可以拥有多个登录会话 |
| ReplayInfo 1:N ReplayFrameInfo | 一条回放记录由多帧状态快照组成 |
