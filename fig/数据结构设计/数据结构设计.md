# 多小车协作巡检仿真系统数据结构设计

本文档按照“字段名、数据类型、数据长度、描述、含义”的形式整理系统主要数据结构。当前项目采用 Redis 黑板保存运行状态，部分结构以 Redis Hash、String、List 和 Stream 的形式存储；表格中的“表”表示逻辑数据表或逻辑数据对象。

## 用户信息表 UserInfo

存储位置：`inspection:auth:users`，Redis Hash，field 为 `username`，value 为用户 JSON。

| 字段名 | 数据类型 | 数据长度 | 描述 | 含义 |
| --- | --- | --- | --- | --- |
| username | varchar | 40 | 主键，不能为空 | 用户账号 |
| displayName | varchar | 40 | 不能为空 | 用户显示名称 |
| role | varchar | 16 | 不能为空 | 用户角色，包含 admin、operator、analyst |
| enabled | boolean | 1 | 不能为空 | 账号是否启用 |
| salt | varchar | 64 | 不能为空 | 密码加盐值 |
| passwordHash | varchar | 128 | 不能为空 | 密码哈希值 |
| createdAt | bigint | 13 | 不能为空 | 创建时间戳 |
| updatedAt | bigint | 13 | 不能为空 | 更新时间戳 |

## 登录会话表 SessionInfo

## 数据结构关系说明

| 关系 | 说明 |
| --- | --- |
| UserInfo 1:N SessionInfo | 一个用户可以拥有多个登录会话 |
| VehicleInfo 1:N TaskInfo | 一辆小车可以在不同时间执行多个任务 |
| FrontierInfo 1:1 TaskInfo | 一个前沿点被分配后生成一个巡检任务 |
| TaskInfo 1:N NavigationRequestInfo | 一个任务可能因为失败或阻塞产生多次导航请求 |
| NavigationRequestInfo 1:1 NavigationPlanInfo | 一个导航请求成功或失败后生成一个规划结果 |
| VehicleInfo 1:1 ComponentHeartbeatInfo | 每辆小车通过心跳上报当前运行状态 |
| ReplayInfo 1:N ReplayFrameInfo | 一条回放记录由多帧状态快照组成 |

## Redis 数据结构汇总

| 逻辑表 | Redis Key | Redis 类型 | 主要用途 |
| --- | --- | --- | --- |
| UserInfo | `inspection:auth:users` | Hash | 保存用户账号和权限 |
| SessionInfo | `inspection:auth:session:{token}` | String | 保存登录会话 |
| RuntimeConfigInfo | `inspection:runtime` | Hash | 保存运行配置 |
| MapInfo | `inspection:map:meta` | Hash | 保存地图元信息 |
| MapChunkInfo | `inspection:map:chunks` | Hash | 保存地图分块 |
| VehicleInfo | `inspection:vehicles` | Hash | 保存小车状态 |
| FrontierInfo | `inspection:frontiers` | Hash | 保存前沿点 |
| TaskInfo | `inspection:tasks` | Hash | 保存巡检任务 |
| NavigationRequestInfo | `inspection:navigation_requests` | Hash | 保存导航请求 |
| NavigationPlanInfo | `inspection:navigation_plans` | Hash | 保存路径规划结果 |
| ComponentHeartbeatInfo | `inspection:heartbeats` | Hash | 保存组件心跳 |
| EventLogInfo | `inspection:events` | Stream | 保存系统事件 |
| ReplayInfo | `inspection:replays:index` | Hash | 保存回放索引 |
| ReplayFrameInfo | `inspection:replays:frames:{replayId}` | List | 保存回放帧 |
