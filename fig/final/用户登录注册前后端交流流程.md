# 用户登录与注册前后端交流流程

本文只整理用户认证相关逻辑。项目中没有普通用户自己注册的公开接口，“注册/新增用户”是超级管理员登录后在用户管理页创建账号。下面所有代码位置都写成“文件名 + 行号 + 方法名/接口名”，方便对着代码查。

## 二、登录流程

整体链路如下：

```text
用户填写账号密码
-> LoginView.vue:submit()
-> api.js:api.login(...) / login(username, password)
-> main.py:login(...) / POST /auth/login
-> auth_store.py:login(username, password)
-> auth_store.py:hash_password(...) / verify_password(...)
-> api.js:setToken(token)
-> App.vue:handleLoggedIn(user)
-> App.vue 根据 user.role 切换页面
```

1. 前端显示登录表单  
   `LoginView.vue:14-39` 是登录表单，使用 `<form @submit.prevent="submit">`。点击登录时不会刷新页面，而是进入 `LoginView.vue:58-68` 的 `submit()` 方法。

2. 前端发起登录请求  
   `LoginView.vue:62` 的 `submit()` 调用 `api.login(this.username, this.password)`。  
   `api.js:167-170` 的 `api.login(...)` 内部调用 `api.js:40-44` 的 `login(username, password)`，发送：

```text
POST /auth/login
body: { username, password }
```

3. 后端接收账号密码  
   `main.py:356-365` 的 `login(...)` 是 `POST /auth/login` 接口。它从请求体中取出 `username` 和 `password`，然后调用 `auth_store.login(...)`。如果校验失败，返回 `401 账号或密码错误`。

4. 后端校验密码并创建会话  
   `auth_store.py:92-106` 的 `login(username, password)` 先读取用户，再检查用户是否存在、是否启用、密码是否正确。  
   密码校验使用 `auth_store.py:35-48` 的 `hash_password(...)` 和 `verify_password(...)`，不会直接明文比较。  
   登录成功后，后端生成随机 token，并用 Redis `setex` 写入 session，会话有效期是 12 小时，最后返回：

```json
{
  "token": "...",
  "user": {
    "username": "...",
    "displayName": "...",
    "role": "...",
    "enabled": true
  }
}
```

5. 前端保存登录状态  
   `api.js:167-170` 的 `api.login(...)` 登录成功后调用 `setToken(result.token)`。  
   `api.js:3-11` 的 `getToken()` / `setToken(token)` 使用 `inspection-auth-token` 作为 key，把 token 存入浏览器 `localStorage`。

6. 页面根据角色跳转  
   `LoginView.vue:63` 登录成功后触发 `logged-in` 事件。  
   `App.vue:68-70` 的 `handleLoggedIn(user)` 接收 user 后保存到 `this.user`。  
   `App.vue:10-18` 的模板根据角色显示页面：`admin` 进入用户管理，`operator` 进入仿真运行，其他角色进入回放分析。

## 三、已登录后的鉴权流程

前端后续每次请求都会走 `api.js:14-38` 的 `request(path, options)`。它会通过 `getToken()` 取出 token，并在 `api.js:17-20` 自动加上请求头：

```text
Authorization: Bearer <token>
```

后端统一鉴权在 `main.py:110-134` 的 `authorize_request(...)`。它会跳过 `/auth/login` 等公开路径；其他接口会调用 `main.py:105-107` 的 `bearer_token(request)` 从请求头取 token，再调用 `auth_store.get_session(...)` 查询会话。

`auth_store.py:108-118` 的 `get_session(token)` 会根据 token 读取 Redis 中的 session，再查出用户信息。如果 token 不存在、过期、用户被停用，后端返回未登录。

刷新页面时也会恢复登录状态：`App.vue:54-65` 的 `created()` 如果本地有 token，就调用 `api.currentUser()`；对应前端方法是 `api.js:47-49` 的 `fetchCurrentUser()`，后端接口是 `main.py:368-370` 的 `current_user(...)`。

## 四、退出登录流程

```text
用户点击退出
-> App.vue:handleLogout()
-> api.js:api.logout()
-> api.js:logout()
-> main.py:logout(...) / POST /auth/logout
-> auth_store.py:logout(token)
-> api.js:setToken("")
-> App.vue 把 user 置空，回到登录页
```

具体代码位置：

- `App.vue:71-76` 的 `handleLogout()`：点击退出后调用 `api.logout()`，最后把 `this.user` 设为 `null`。
- `api.js:173-180` 的 `api.logout()`：内部调用 `api.js:51-52` 的 `logout()` 发送 `POST /auth/logout`，最后清除 token。
- `main.py:373-376` 的 `logout(...)`：后端调用 `auth_store.logout(...)`。
- `auth_store.py:120-122` 的 `logout(token)`：删除 Redis 里的 `session:<token>`。

## 五、注册/新增用户流程

这里的“注册”不是开放注册，而是管理员新增用户。

```text
管理员登录
-> App.vue 根据 role === 'admin' 显示 AdminUsers.vue
-> AdminUsers.vue:openCreate()
-> AdminUsers.vue:save()
-> api.js:createUser(payload)
-> main.py:create_user(...) / POST /admin/users
-> auth_store.py:create_user(...)
-> AdminUsers.vue:loadUsers() 重新加载用户列表
```

1. 只有管理员能进入用户管理  
   `App.vue:10-13` 中只有 `user.role === 'admin'` 时显示 `AdminUsers.vue`。  
   后端也有二次校验：`main.py:126-127` 的 `authorize_request(...)` 限制 `/admin/` 接口只有超级管理员能访问。

2. 管理员填写新增用户表单  
   `AdminUsers.vue:57-89` 是新增/编辑用户弹窗，包含用户名、姓名、密码、角色、是否允许登录。  
   `AdminUsers.vue:145-149` 的 `openCreate()` 点击“新增用户”时打开空表单。

3. 前端提交新增用户  
   `AdminUsers.vue:164-185` 的 `save()` 组装 `username`、`displayName`、`password`、`role`、`enabled`。新增时调用 `api.createUser(payload)`。  
   `api.js:59-60` 的 `createUser(payload)` 对应发送：

```text
POST /admin/users
body: { username, displayName, password, role, enabled }
```

4. 后端创建用户  
   `main.py:384-396` 的 `create_user(...)` 接收新增用户请求，并调用 `auth_store.create_user(...)`。  
   `auth_store.py:131-161` 的 `create_user(...)` 校验用户名格式、密码长度、角色范围和用户名是否重复，然后对密码做哈希，最后把用户写入 Redis。返回给前端的用户信息会去掉 `salt` 和 `passwordHash`。

5. 前端刷新列表  
   新增成功后，`AdminUsers.vue:180-185` 的 `save()` 显示“用户已创建”，关闭弹窗，并调用 `loadUsers()`。  
   `AdminUsers.vue:128-134` 的 `loadUsers()` 调用 `api.listUsers()`；对应前端方法是 `api.js:55-56` 的 `fetchUsers()`，后端接口是 `main.py:379-381` 的 `list_users()`。

## 六、补充说明

系统启动时会自动准备三个默认账号：`auth_store.py:73-76` 的 `ensure_super_admin()` 创建超级管理员、系统运行员和分析员。管理员后续创建的新用户只能是 `operator` 或 `analyst`，这一点由 `auth_store.py:144-145` 的 `create_user(...)` 限制。
