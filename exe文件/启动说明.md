# 多小车协作巡检系统 exe 启动说明

## 构件清单

- `api_gateway.exe`：API 网关/接口服务，提供 HTTP、WebSocket、静态前端资源访问。
- `controller_worker.exe`：控制器计算组件，操作系统层面只允许一个实例，同时使用 Redis 控制器锁。
- `navigator_worker.exe`：导航器计算组件，单个进程可托管 1 到 N 个逻辑导航器。
- `robot_worker.exe`：小车执行/仿真计算组件，推进车辆移动、扫描和地图更新。
- `display_ui.exe`：独立显示界面组件，可启动多个实例同时观察系统状态。

## 启动顺序

先启动 Redis，默认地址为：

```powershell
redis://127.0.0.1:6379/0
```

然后在本目录依次启动：

```powershell
.\api_gateway.exe --host 127.0.0.1 --port 8000
.\controller_worker.exe
.\navigator_worker.exe
.\robot_worker.exe
.\display_ui.exe --port 8081 --api-url http://127.0.0.1:8000
```

浏览器访问：

```text
http://127.0.0.1:8081/pathfinding
```

## 常用参数

```powershell
.\navigator_worker.exe
.\navigator_worker.exe
.\navigator_worker.exe
.\robot_worker.exe --steps-per-tick 2
.\display_ui.exe --port 8082 --api-url http://127.0.0.1:8000
```

`display_ui.exe` 可多开；如果端口被占用，会自动尝试后续端口。`controller_worker.exe` 多开时，后启动的实例会自动退出。
